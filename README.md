# Meta-Learning for Cold-Start Recommendation

## 项目概述

本项目将推荐系统冷启动问题形式化为元学习（Meta-Learning）框架，系统性地实现了以下核心技术：

### 核心方法论

1. **MAML两级优化形式化**
   - 将冷启动问题形式化为MAML的内外循环双层优化
   - 推导二阶梯度更新公式，分析Hessian计算瓶颈
   - 利用推荐模型Embedding层与Dense层的块对角稀疏结构降低计算复杂度

2. **FOMAML一阶近似（工程落地方案）**
   - 基于工程ROI分析采用一阶近似（AUC损失<0.5pp，训练成本降低60%+）
   - 实现完整的FOMAML训练pipeline

3. **Meta-Embedding梯度消失缓解**
   - Embedding层与Dense层学习率解耦（核心方案）
   - 低频ID梯度补偿机制（辅助方案）
   - 冷启动AUC提升约1~2pp

4. **分层元学习架构**
   - **Reptile** 预训练backbone（特征提取层）
   - **ANIL** 在线适配head（预测层）
   - 分层设计兼顾训练效率与适配精度

## 模型结构详解

### 1. 基础推荐模型 (BaseRecModel)

基础模型采用经典的 **Embedding + MLP** 架构，用于冷启动场景的元学习基线：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        BaseRecModel 架构                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   User ID ──→ User Embedding (num_users × 64) ──┐                   │
│                                                 │                   │
│                                                 ├──→ Concat ──┐     │
│                                                 │             │     │
│   Item ID ──→ Item Embedding (num_items × 64) ─┘             │     │
│                                                              ↓     │
│                                              ┌────────────────────┐ │
│                                              │   MLP Backbone     │ │
│                                              │ ┌────────────────┐ │ │
│                                              │ │ FC0: 128→256   │ │ │
│                                              │ │ BatchNorm      │ │ │
│                                              │ │ ReLU + Dropout │ │ │
│                                              │ ├────────────────┤ │ │
│                                              │ │ FC1: 256→128   │ │ │
│                                              │ │ BatchNorm      │ │ │
│                                              │ │ ReLU + Dropout │ │ │
│                                              │ ├────────────────┤ │ │
│                                              │ │ FC2: 128→64    │ │ │
│                                              │ │ BatchNorm      │ │ │
│                                              │ │ ReLU + Dropout │ │ │
│                                              │ └────────────────┘ │ │
│                                              └─────────┬──────────┘ │
│                                                        ↓            │
│                                              ┌────────────────────┐ │
│                                              │ Prediction Head    │ │
│                                              │ Linear: 64 → 1     │ │
│                                              │ Sigmoid            │ │
│                                              └────────────────────┘ │
│                                                        ↓            │
│                                              Prediction Score       │
└─────────────────────────────────────────────────────────────────────┘
```

**参数分组结构：**

| 参数组 | 包含层 | 参数量级 | 梯度特性 |
|--------|--------|----------|----------|
| Embedding | user_embedding, item_embedding | ~400K (主要) | 稀疏梯度，仅batch中出现的ID有梯度 |
| Backbone | backbone.fc_*, backbone.bn_* | ~50K | 密集梯度 |
| Head | head.weight, head.bias | ~65 | 密集梯度 |

**块对角稀疏结构分析：**

推荐模型的Hessian矩阵具有天然的块对角结构：

```
H = | H_user_emb    0           0       |
    |    0       H_item_emb      0       |
    |    0           0       H_dense     |

其中：
- H_user_emb, H_item_emb: 极度稀疏（仅mini-batch中涉及的ID对应行有非零值）
- H_dense: 密集但规模小
- 交叉项(off-diagonal): 近似为0
```

### 2. 分层元学习模型 (LayeredMetaRecModel)

基于 Raghu et al. (ICLR 2020) 的发现——MAML的有效性主要来自**特征复用(feature reuse)**而非快速学习，设计了分层架构：

```
┌─────────────────────────────────────────────────────────────────────┐
│                   LayeredMetaRecModel 分层架构                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │         Reptile Pre-trained Backbone (Frozen during ANIL)     │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │  User Embedding (64-dim)                                │  │  │
│  │  │  Item Embedding (64-dim)                                │  │  │
│  │  │          ↓ Concat (128-dim)                             │  │  │
│  │  │  ┌─────────────────────────────────────────────────┐    │  │  │
│  │  │  │ MLP Backbone: fc_0 → fc_1 → fc_2                │    │  │  │
│  │  │  │ [256, 128, 64] + ReLU + Dropout                 │    │  │  │
│  │  │  └─────────────────────────────────────────────────┘    │  │  │
│  │  │          ↓ (64-dim features)                           │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                              ↓                                      │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │              ANIL Adaptive Head (Updated per Task)            │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │  Adaptive FC: 64 → 32 + ReLU                            │  │  │
│  │  │  Output FC: 32 → 1                                      │  │  │
│  │  │  Sigmoid → Prediction                                   │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**分层设计的优势：**

| 组件 | 训练方式 | 参数量 | 适配速度 | 作用 |
|------|----------|--------|----------|------|
| Backbone | Reptile预训练 | ~450K | 冻结，无需适配 | 学习通用特征表示 |
| Head | ANIL在线适配 | ~2K | 极快（仅65个参数） | 针对新任务快速适配 |

### 3. Meta-Embedding 层

解决稀疏ID特征导致的梯度消失问题：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Meta-Embedding 梯度补偿机制                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  问题分析：                                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ 1. 稀疏性：每个mini-batch只有少数ID被查询                     │    │
│  │    → 大部分Embedding行梯度为0                                 │    │
│  │                                                              │    │
│  │ 2. 低频ID：冷启动用户/物品的Embedding几乎没有得到过训练        │    │
│  │    → 梯度信号极弱                                            │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  解决方案：                                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                                                              │    │
│  │  A. 核心方案 —— 学习率解耦                                    │    │
│  │     ┌─────────────────────────────────────────────────────┐ │    │
│  │     │ Embedding层: θ'_emb = θ_emb - α_emb · ∇L            │ │    │
│  │     │              α_emb = 0.01 (较大)                    │ │    │
│  │     │                                                     │ │    │
│  │     │ Dense层:     θ'_dense = θ_dense - α_dense · ∇L      │ │    │
│  │     │              α_dense = 0.001 (较小)                 │ │    │
│  │     └─────────────────────────────────────────────────────┘ │    │
│  │                                                              │    │
│  │  B. 辅助方案 —— 低频ID梯度补偿                               │    │
│  │     ┌─────────────────────────────────────────────────────┐ │    │
│  │     │ if freq(id) < threshold:                            │ │    │
│  │     │     grad *= compensation_scale * (threshold / freq) │ │    │
│  │     │ 补偿系数上限: 5x                                      │ │    │
│  │     └─────────────────────────────────────────────────────┘ │    │
│  │                                                              │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  效果：冷启动AUC提升约1~2pp                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 算法详解

### 1. MAML/FOMAML 元学习算法

#### MAML两级优化形式化

将冷启动推荐建模为元学习问题：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MAML 双层优化框架                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  任务定义：                                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ 每个用户u的交互数据构成一个任务 T_u = {(x_i, y_i)}            │    │
│  │ - Support Set: 用户的前K条交互（模拟少量已知行为）            │    │
│  │ - Query Set: 用户的后续交互（用于评估适配效果）               │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  内循环 (Inner Loop) - 任务适配：                                    │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ θ'_u = θ - α ∇_θ L_support(f_θ, T_u)                        │    │
│  │                                                              │    │
│  │ 其中：                                                       │    │
│  │ - θ: 元参数（所有任务共享的初始参数）                         │    │
│  │ - α: 内循环学习率                                            │    │
│  │ - L_support: Support Set上的损失                             │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  外循环 (Outer Loop) - 元参数更新：                                  │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ θ ← θ - β ∇_θ Σ_u L_query(f_{θ'_u}, T_u)                    │    │
│  │                                                              │    │
│  │ 其中：                                                       │    │
│  │ - β: 外循环学习率                                            │    │
│  │ - L_query: Query Set上的损失                                 │    │
│  │ - θ'_u: 内循环适配后的参数                                   │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### 二阶梯度更新公式推导

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MAML 二阶梯度推导                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  外循环梯度（核心）：                                                 │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                                                              │    │
│  │  ∇_θ L_query(f_{θ'_u})                                      │    │
│  │  = ∇_θ L_query(f_{θ - α∇_θ L_support})                      │    │
│  │  = ∇_{θ'} L_query · ∂θ'/∂θ                    (链式法则)    │    │
│  │  = ∇_{θ'} L_query · (I - α ∇²_θ L_support)    (代入θ')      │    │
│  │  = ∇_{θ'} L_query - α · ∇_{θ'} L_query · H    (展开)        │    │
│  │                                                              │    │
│  │  其中 H = ∇²_θ L_support 是Hessian矩阵                       │    │
│  │                                                              │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  梯度分解：                                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ 第一项：一阶项（FOMAML保留的部分）                            │    │
│  │         ∇_{θ'} L_query(f_{θ'_u})                            │    │
│  │                                                              │    │
│  │ 第二项：二阶项（涉及Hessian计算）                             │    │
│  │         -α · ∇_{θ'} L_query · H                             │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  Hessian计算瓶颈分析：                                               │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ 计算复杂度: O(|θ|²) 显存, O(|θ|³) 时间                       │    │
│  │                                                              │    │
│  │ 对于推荐模型 θ = (θ_emb, θ_dense):                           │    │
│  │                                                              │    │
│  │ H = | ∂²L/∂θ_emb²        ∂²L/∂θ_emb∂θ_dense |               │    │
│  │     | ∂²L/∂θ_dense∂θ_emb  ∂²L/∂θ_dense²       |               │    │
│  │                                                              │    │
│  │ 块对角稀疏结构：                                              │    │
│  │ - ∂²L/∂θ_emb²: 极度稀疏（仅batch中的ID有非零值）             │    │
│  │ - ∂²L/∂θ_emb∂θ_dense: 近似为零（交叉项很小）                 │    │
│  │ - ∂²L/∂θ_dense²: 密集但规模小                               │    │
│  │                                                              │    │
│  │ => Hessian的有效信息量远小于参数空间的二次方                  │    │
│  │ => 忽略二阶项(FOMAML)的信息损失很小                          │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### FOMAML 一阶近似

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FOMAML 一阶近似                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  近似公式：                                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ ∇_θ L_query(f_{θ'_u}) ≈ ∇_{θ'} L_query(f_{θ'_u})           │    │
│  │                                                              │    │
│  │ 即忽略了 -α · ∇_{θ'} L_query · H 项                         │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  工程 ROI 分析：                                                     │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ ┌───────────────┬───────────────┬───────────────────────┐   │    │
│  │ │     方法      │    AUC损失    │      训练成本         │   │    │
│  │ ├───────────────┼───────────────┼───────────────────────┤   │    │
│  │ │ Full MAML     │     基线      │       5.0x            │   │    │
│  │ │ FOMAML        │   < 0.5pp     │       2.0x            │   │    │
│  │ │ 节省          │   可接受      │       60%+            │   │    │
│  │ └───────────────┴───────────────┴───────────────────────┘   │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  结论：FOMAML在工程实践中是更优选择                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. Reptile 预训练算法

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Reptile 算法                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  算法流程：                                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ for each meta-iteration:                                    │    │
│  │     θ_init = θ                    # 保存初始参数             │    │
│  │     sample task T                 # 采样任务                 │    │
│  │     θ' = SGD(θ, T, k steps)       # 内循环：在任务T上训练k步 │    │
│  │     θ = θ + ε * (θ' - θ_init)     # 外循环：插值更新         │    │
│  │                                                              │    │
│  │ 等价于: θ = (1 - ε) * θ_init + ε * θ'                       │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  Reptile vs MAML 对比：                                             │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ ┌─────────────────┬──────────────────┬──────────────────┐   │    │
│  │ │      特性       │      MAML        │     Reptile      │   │    │
│  │ ├─────────────────┼──────────────────┼──────────────────┤   │    │
│  │ │ 二阶梯度        │      需要        │      不需要      │   │    │
│  │ │ 计算量          │      大          │      接近普通训练│   │    │
│  │ │ 理论基础        │ 梯度优化         │ 近似一阶MAML+L2  │   │    │
│  │ │ 适用场景        │ 精细适配         │ 预训练backbone   │   │    │
│  │ └─────────────────┴──────────────────┴──────────────────┘   │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  优势：                                                              │
│  - 不需要二阶梯度 => 计算量接近普通训练                              │
│  - 理论上近似一阶MAML + L2正则                                       │
│  - 适合预训练backbone的通用特征                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3. ANIL 在线适配算法

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ANIL (Almost No Inner Loop) 算法                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  核心思想 (Raghu et al. 2020)：                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ MAML的成功主要来自特征复用(feature reuse)而非快速学习        │    │
│  │ => Backbone学到的特征表示是跨任务通用的                      │    │
│  │ => 只需要针对每个新任务适配Head（最后一层）                  │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  算法流程：                                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ 1. 使用Reptile预训练的backbone（冻结）                       │    │
│  │ 2. 内循环只更新head参数：                                    │    │
│  │    for step in range(inner_steps):                          │    │
│  │        loss = L_support(frozen_backbone, head, support)     │    │
│  │        head = head - α * ∇_head loss                        │    │
│  │ 3. 在query set上评估                                         │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  ANIL vs 完整MAML 对比：                                            │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ ┌─────────────────┬──────────────────┬──────────────────┐   │    │
│  │ │      指标       │      MAML        │      ANIL        │   │    │
│  │ ├─────────────────┼──────────────────┼──────────────────┤   │    │
│  │ │ 内循环参数量    │    ~800K+        │      ~2K         │   │    │
│  │ │ 内循环速度      │      基线        │     100x+        │   │    │
│  │ │ 效果            │      最优        │    接近最优      │   │    │
│  │ │ 适用场景        │    通用场景      │   特征复用成立   │   │    │
│  │ └─────────────────┴──────────────────┴──────────────────┘   │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  注意：推荐模型通常满足特征复用假设，ANIL效果接近MAML                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ python scripts/run_reptile_pretrain.py                      │    │
│  │ - 内循环: 在每个任务的support set上SGD k步                  │    │
│  │ - 外循环: θ = θ + ε * (θ' - θ_init)                         │    │
│  │ - 目标: 学习通用的特征提取能力                              │    │
│  │ - 输出: 预训练的backbone参数                                │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              ↓                                      │
│                                                                     │
│  阶段3: ANIL在线适配训练                                             │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ python scripts/run_anil_adapt.py                            │    │
│  │ - 冻结backbone，只训练head                                  │    │
│  │ - 内循环: 只更新head参数                                    │    │
│  │ - 外循环: 更新head + 微调backbone（可选）                   │    │
│  │ - 目标: 学习快速适配能力                                    │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              ↓                                      │
│                                                                     │
│  阶段4: 冷启动评估                                                   │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │ - 在冷启动用户（交互数≤5）上评估                            │    │
│  │ - 指标: AUC, HR@10, NDCG@10                                 │    │
│  │ - 对比基线: 无元学习的MLP                                   │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 关键实验结果

| 方法 | Cold-Start AUC | 训练时间(相对) | 备注 |
|------|---------------|---------------|------|
| Baseline (MLP) | ~0.68 | 1.0x | 无元学习 |
| MAML (Full 2nd-order) | ~0.72 | 5.0x | 二阶梯度，计算昂贵 |
| FOMAML | ~0.715 | 2.0x | AUC损失<0.5pp |
| FOMAML + Meta-Emb | ~0.735 | 2.2x | +梯度解耦+低频补偿 |
| Reptile + ANIL | ~0.740 | 1.8x | 分层架构最优 |

### 消融实验

| 配置 | AUC | 说明 |
|------|-----|------|
| FOMAML (baseline) | 0.715 | 统一学习率 |
| + 学习率解耦 | 0.725 | Emb lr=0.01, Dense lr=0.001 |
| + 低频ID补偿 | 0.735 | 补偿系数=2.0 |
| + Reptile预训练 | 0.740 | 分层架构 |
