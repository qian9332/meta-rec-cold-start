## 五、训练数据加工链路

### 5.1 数据处理6步流程

```
原始数据 (1,000,209评分)
    │
    ▼ Step 1: ID重映射
UserID 1~6040 → 0~6039,  MovieID 1~3952 → 0~3705
    │
    ▼ Step 2: 标签二值化
rating ≥ 4 → label=1 (正样本57.5%)
rating < 4 → label=0 (负样本42.5%)
    │
    ▼ Step 3: 交互数过滤 (20 ≤ n ≤ 200)
6,040用户 → 4,462用户 (73.9%)   过滤掉1,578个高活跃用户(>200条)
    │
    ▼ Step 4: 随机采样 (seed=42)
4,462用户 → 训练100 + 评估30 = 130用户
    │
    ▼ Step 5: 时序切分 → Meta-Learning Task
每用户按时间排序 → 前5条=support(冷启动) + 后续=query(泛化)
    │
    ▼ Step 6: 负采样 (1:3)
每个正样本配3个随机负样本 → support 20条 + query 80条 = 每task 100条
```

## 六、冷启动模拟分析

### 6.1 冷启动定义

- 每用户只取**前5条交互**作为 support set → 模拟新用户只有5次行为
- 后续交互作为 query set → 评估模型泛化能力

### 6.2 Support Set正样本分布

| 前5条中正样本数 | 用户数 | 占比 |
|:---:|:---:|:---:|
| 0 | 6 | 4.6% |
| 1 | 15 | 11.5% |
| 2 | 34 | 26.2% |
| 3 | 30 | 23.1% |
| 4 | 27 | 20.8% |
| 5 | 18 | 13.8% |

→ **4.6%的用户前5条全是负样本**，这对元学习的support set质量提出挑战

### 6.3 低频ID统计（梯度补偿机制验证）

| 指标 | 值 |
|------|-----|
| 训练涉及item数 | 1,844 |
| item频次最小值 | 1 |
| item频次最大值 | 50 |
| item频次平均值 | 4.10 |
| item频次中位数 | 2 |
| **低频item(≤3次)** | **1,199 (65.0%)** |
| **单次item(=1次)** | **641 (34.8%)** |

→ **65%的item是低频(≤3次交互)**，34.8%仅出现1次  
→ 这些低频ID的Embedding在训练中几乎得不到梯度更新 → **验证了梯度补偿机制的必要性**

## 七、数据集与项目技术点适配总结

| 项目技术点 | 数据集支撑 | 适配度 |
|:---|:---|:---:|
| MAML两级优化 | 每用户=1个task, support/query自然切分 | ⭐⭐⭐⭐⭐ |
| Hessian块对角稀疏 | user/item/age/gender/occupation多Embedding层 | ⭐⭐⭐⭐⭐ |
| FOMAML一阶近似 | 足够数据量对比二阶/一阶差异 | ⭐⭐⭐⭐⭐ |
| Meta-Embedding梯度消失 | 低频occupation(farmer=17人)天然存在 | ⭐⭐⭐⭐⭐ |
| 步长解耦 | Embedding层 vs Dense层天然分离 | ⭐⭐⭐⭐⭐ |
| 低频ID梯度补偿 | 65%低频item(≤3次), 34.8%单次item | ⭐⭐⭐⭐⭐ |
| Reptile预训练backbone | 4,462个用户task提供丰富预训练先验 | ⭐⭐⭐⭐ |
| ANIL在线适配head | 新用户到来仅微调head,5条交互即可适配 | ⭐⭐⭐⭐⭐ |
