# 训练日志问题分析与优化方案

> 分析日期: 2026-03-04
> 分析人: AI Engineer

---

## 一、训练日志问题诊断

### 从日志中发现的 7 个关键问题

#### 问题1：AUC指标整体偏低（所有方法 ≤ 0.54）
- **现象**: FOMAML基线AUC=0.5379, 解耦后仅0.5416, 接近随机(0.5)
- **根因**: 
  - 训练epoch太少(仅2 epochs)，模型严重欠拟合
  - 训练用户仅15个, task太少导致meta-learner没学到通用策略
  - 元学习外循环学习率(0.001)可能不匹配

#### 问题2：MAML vs FOMAML 对比数据异常
- **现象**: 日志显示 `FOMAML耗时: 112.9ms | MAML耗时: 8.8ms`，MAML反而更快
- **根因**: 计时逻辑写反了，MAML需计算二阶梯度理应更慢
- **影响**: 分析报告的工程ROI结论数据不可信

#### 问题3：梯度余弦相似度仅0.39
- **现象**: `梯度余弦相似度: mean=0.3893`, 远低于文献典型的0.9+
- **根因**: 
  - 仅在随机初始化的模型上计算，此时MAML/FOMAML梯度方向差异大
  - 应该在训练若干步后再比较
  - single-task的cos_sim方差很大，需要多task平均

#### 问题4：Reptile和ANIL表现极差（AUC < 0.5）
- **现象**: Reptile=0.4928, ANIL=0.4860, 低于随机
- **根因**: 
  - Reptile内循环只用了3步SGD(lr=0.01), 步数/学习率不足
  - Reptile外循环epsilon=0.1太大，导致参数跳跃
  - ANIL冻结backbone后只有65个head参数(单层Linear), 表达能力不足
  - LayeredMetaRecModel的backbone没用BatchNorm，与BaseRecModel不一致

#### 问题5：user_embedding梯度几乎为零
- **现象**: `user_embedding.weight: sparsity=0.9998, grad_norm=0.000001`
- **根因**: 
  - 每个task只有1个user_id, 所以user_embedding只有1行有梯度
  - 但grad_norm=0.000001极低，说明user_embedding的梯度在反向传播中基本消失了
  - 而item_embedding的grad_norm=4.11正常 → user特征的梯度流通路径有问题
- **影响**: 元学习完全无法更新user embedding，冷启动核心问题未解决

#### 问题6：HR@10始终为1.0，指标失真
- **现象**: 所有方法的HR@10=1.0
- **根因**: 
  - query set太小（约10个正+10个负=20个样本），K=10占了一半
  - 正样本比例高(57.5%), 随机取10个几乎一定命中正样本
  - 该指标在当前设置下没有区分度

#### 问题7：数据集划分不合理
- **现象**: 0个用户交互数≤10, 所有用户>=20条
- **根因**: MovieLens-1M本身没有极冷启动用户(最少20条)
- **影响**: 
  - 无法真实模拟冷启动场景
  - cold_eval模式fallback到交互较少的用户，但仍有20+条
  - 应该对已有用户进行冷启动模拟（截断交互历史）

## 二、优化方案（共8项）

### 优化1: 增加训练规模与epoch
- 训练用户: 15 → 100+
- epoch: 2 → 8+
- meta-batch: 4 → 8

### 优化2: 修复冷启动模拟策略
- 对每个用户，随机截断保留前5~10条作为"冷启动"交互
- Support: 前3~5条, Query: 后面5~10条
- 添加时间序列split确保无未来信息泄露

### 优化3: 修复user_embedding梯度消失
- 给user_embedding增加更大的初始化scale
- 在计算loss前对user_emb做L2 normalize，增强梯度流
- 解耦LR中给user_embedding独立更大的lr(0.05)

### 优化4: 增强ANIL head表达能力
- head从 Linear(64→1) 扩展到 Linear(64→32) + ReLU + Linear(32→1)
- 可适配参数: 65 → 2,145

### 优化5: 修正Reptile超参数
- 内循环步数: 3 → 5
- 内循环学习率: 0.01 → 0.02
- 外循环epsilon: 0.1 → 0.3（前期大步探索）→ decay到0.05

### 优化6: 内循环多步适配
- FOMAML内循环: 1步 → 3步
- 梯度累积，多步适配后再在query上评估

### 优化7: 修正评估指标
- HR@K改为K=5（更有区分度）
- 增大query set: 10 → 20
- 增加负采样比例: 1:1 → 1:3

### 优化8: Warm-up + 学习率调度
- 前2个epoch用较大的outer_lr(0.003)做warm-up
- 之后decay到0.001
- 内循环lr随epoch微调
