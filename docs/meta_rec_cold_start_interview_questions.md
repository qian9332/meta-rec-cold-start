## 第一部分：数据处理与冷启动场景构建（难度：⭐⭐⭐）

### Q1.1：为什么选择MovieLens-1M数据集？这个数据集在冷启动研究中有哪些局限性？

**答案**：

**选择原因**：
1. **经典基准**：学术界广泛使用，便于结果对比
2. **规模适中**：100万评分，适合快速实验迭代
3. **数据质量**：用户行为真实，评分分布合理
4. **易于获取**：公开可用，无版权限制

**局限性**：
1. **场景单一**：仅包含电影领域，缺乏跨域泛化验证
2. **交互稀疏**：冷启动用户（≤5交互）比例相对较小，可能不充分
3. **无序列信息**：原始数据缺少时间戳排序，损失了行为序列模式
4. **无内容特征**：缺少用户画像、电影描述等side information
5. **评分偏置**：评分数据而非隐式反馈，与实际推荐场景有gap
6. **数据老化**：数据收集于2000年，无法反映现代用户行为模式

**改进方向**：
- 使用更大规模数据集（如Amazon、Taobao）
- 引入跨域数据验证泛化能力
- 添加序列建模任务
- 融合内容特征
- 使用隐式反馈数据集

### Q1.2：冷启动用户定义为"交互数≤5"，这个阈值如何确定？在实际业务中如何调整？

**答案**：

**阈值确定依据**：
1. **数据分布分析**：统计用户交互数的分布，找到"长尾"起点
2. **经验法则**：常见阈值有3、5、10，需根据业务调整
3. **任务难度平衡**：阈值太小样本太少，太大则不是真冷启动
4. **业务场景考虑**：不同场景的用户行为频率不同

**实际业务调整策略**：

1. **分阶段定义**：
   - 0交互：绝对冷启动
   - 1-3交互：早期冷启动
   - 3-10交互：中期冷启动

2. **业务场景依赖**：
   - 高频应用（新闻、短视频）：≤3交互
   - 中频应用（电商）：≤5交互
   - 低频应用（旅游、房产）：≤10交互

3. **动态调整代码示例**：
```python
def get_cold_start_threshold(interactions_data):
    # 方法1：基于分位数
    threshold = np.percentile(user_interaction_counts, 10)
    
    # 方法2：基于拐点检测
    from kneed import KneeLocator
    x = sorted(user_interaction_counts)
    y = np.arange(len(x)) / len(x)
    knee = KneeLocator(x, y, curve='concave')
    threshold = knee.knee
    
    return max(3, min(10, int(threshold)))
```

4. **实验验证**：
   - 对比不同阈值下的模型性能
   - 考虑样本量和任务难度的平衡
